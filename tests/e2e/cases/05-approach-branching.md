# 05-approach-branching: 真分叉时 2-3 方案 + 领头推荐

**规则**（来自 `prompts.ts` "方案探索"节）：
- 进入 stage 4（`generate_solution`）且存在多种合理方案时，AI 主动提 2-3 方案
- 领头放推荐项并明确说"我更倾向 A，因为..."
- 呈现方式是对话式，不是纯列表
- 如无真分叉则直说"我想这样做..."，不硬凑多方案

## 前置

- 干净加载

## 步骤

1. 用自由文字引导对话推进到 stage 4 附近。发送依次：
   - 第 1 轮：`我想做一个员工培训报名系统。公司 HR 用，给所有正式员工开放报名`
   - 第 2 轮：AI 追问时用一句话简洁作答（比如 `核心流程：HR 发课程、员工看列表报名、HR 看名单；数据量一个月也就几十门课`）
   - 第 3 轮：AI 继续追问时再答 `约束是必须能和现有 OA 的账号体系对接，SSO 登录`
   - 第 4 轮：若 AI 问到方案/界面选择，答 `你先给我几个方案看看好不好`
2. 观察 AI 最新回复与进度条 stage
3. 如 stage 已到 `generate_solution`：
   - 读取气泡文字
   - 若含选项卡，读取所有选项的 label + description

## 期望

当 AI 明确进入"给方案"阶段时（判定：气泡文字含"方案"、"思路"、"选项"之一，或进度条 stage=generate_solution）：

- 气泡文字字符数 ≥ 80（方案讨论需要一定篇幅）
- 满足下列之一（体现"2-3 方案 + 推荐"结构）：
  - (a) 选项卡数量 2-3 个，且第一个选项 label 或 description 含"（推荐）"字样 **或** 气泡文字明确指出"我更倾向"/"我推荐"类表述指向其中一项
  - (b) 无选项卡但气泡文字分点列出了 2-3 个方案，且明确表达了推荐意图（含"我更倾向"、"推荐"、"建议先"等之一）
- 不应出现"一个方案也没给、只是继续追问更多背景"的情况（步骤 2 已喂足了背景）

## 通过标准

到达 stage 4 时给出 2-3 方案并带推荐项；可以是选项卡也可以是对话文字。
